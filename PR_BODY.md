## Краткое описание

Добавлено **5 новых туннельных движков**, **система авто-ремедиации**, **единый checker обновлений** и **улучшения UX**. Все изменения **добавочные и обратно совместимые** — существующий функционал не затронут, новые функции отключены по умолчанию.

**24 файла создано, 15 файлов изменено. Нулевые поломки.**

---

## Что добавлено

### 1. WARP/MASQUE туннель (usque-keenetic)
Cloudflare WARP через MASQUE-протокол на порту 443, маскируется под HTTPS. Самый устойчивый к DPI бесплатный VPN.
- Источник: `side-effect-tm/usque-keenetic`
- Паттерн: следует архитектуре существующего `SingboxManager`

### 2. Telegram MTProto туннель (Variant C)
Два движка для разных архитектур:
- `teleproxy` (C) — максимальное сопротивление DPI (fake-TLS, DRS, MSS clamp), только ARM64
- `tg-mtproxy-client` (Go) — все архитектуры включая MIPS
- Автовыбор по архитектуре

### 3. Opera Proxy (Alexey71/opera-proxy)
Zero-config HTTP/SOCKS5 прокси через SurfEasy VPN. Бесплатный fallback когда WARP/AWG заблокированы.

### 4. Auto-Remediation
BlockCheck классифицирует тип DPI → автоматически выбирает метод:
- `TLS_DPI` → strategy scan (nfqws2)
- `IP_BLOCK` → туннель (WARP/AWG/sing-box по настраиваемому приоритету)
- `DNS_FAKE` → DoH/DoT

### 5. Unified Update Checker
Единая проверка обновлений для всех 9 бинарников (zapret2, sing-box, mihomo, AWG, GUI, usque, teleproxy, tgproto, opera). Фоновый процесс с настраиваемым интервалом.

### 6. Расширенные списки доменов
- 12 пресетов (было 6): +TikTok, Netflix, Cloudflare, Google Meet, Russia outside, Ukraine inside
- Per-list транспорт (отдельный транспорт для каждого списка)
- Группировка пресетов по категориям (Сервисы / Страны)
- Кнопка "Добавить в маршрут" на пресетах

### 7. Dashboard объединённый обзор
11 карточек статуса в 3 группах: Ядро, VPN/Туннели, Мониторинг. Все кликабельны.

### 8. Block Detector
DNS-мониторинг + 4-stage probing (DNS → TCP → TLS → HTTP) + авто-добавление заблокированных доменов.

## Обратная совместимость

- Все новые секции конфига имеют `enabled: false` по умолчанию
- Существующие пользователи не заменяют изменений до включения новых функций
- Ни один существующий API-эндпоинт не изменён
- Ни одна существующая страница UI не изменена (только расширена)
- Миграция не требуется — deep-merge автоматически добавляет новые секции

## Тестирование

- Все Python-файлы проходят `ast.parse()` syntax validation
- Все JS-файлы проходят brace-balance validation
- Существующие тесты не сломаны
