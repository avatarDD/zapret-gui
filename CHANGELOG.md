# Changelog

## v0.13.1 — Исправление интерфейсов

### Исправлено
- **Чекбоксы интерфейсов не работают** — убран дублирующий `onclick` на `<label>`, который конфликтовал с `onchange` на `<input>` и приводил к двойному срабатыванию; убран `pointer-events:none` блокировавший клики навсегда; исправлена логика `toggleAllInterfaces()` — теперь корректно переключает между режимами «все» и «конкретные»
- **WAN/WAN6/LAN не заполняются автоматически** — добавлено авто-определение ролей интерфейсов по таблице маршрутов: WAN через `/proc/net/route` и `ip route` (как в zapret2), WAN6 через `ip -6 route`, LAN по bridge-интерфейсу или приватной подсети RFC 1918; значения подставляются в поля при загрузке страницы

### Добавлено
- **Кнопка «Определить WAN/LAN автоматически»** — принудительное авто-определение и заполнение полей WAN/WAN6/LAN
- **Бейджи ролей** — в списке интерфейсов отображаются цветные метки WAN/WAN6/LAN рядом с именем
- **Блок определённых ролей** — информационная панель показывает текущие определённые WAN/WAN6/LAN
- Новое поле `detected` в ответе `GET /api/interfaces` с авто-определёнными ролями
- Placeholder полей WAN/WAN6/LAN показывает определённое значение (`Авто: eth0`)

### Изменено
- Унифицирована версия GUI (`v0.13.1` во всех модулях)

## v0.13.0 — Исправления и улучшения

### Исправлено
- **Логи: ошибки SSE** — устранён спам «Critical error while processing request: /api/logs/stream» при открытии раздела Логи. SSE-генератор вынесен в отдельную функцию с полным перехватом исключений, добавлен JSON-обработчик ошибки 500 для API-маршрутов, на фронтенде ограничено число попыток переподключения
- **Hosts: форма не обновляется** — при добавлении/удалении записей форма теперь автоматически перерисовывается без необходимости обновлять страницу по F5

### Добавлено
- **Подсветка синтаксиса скрипта автозапуска** — shell-скрипт в модальном окне теперь отображается с цветовым выделением: shebang, комментарии, строки, переменные, ключевые слова, команды (iptables/nfqws2/echo/grep), пути, флаги, числа, операторы
- **Uptime в человекочитаемом формате** — вместо секунд отображается «2мес 5д 3ч 42м 15с»
- **Выбор сетевых интерфейсов** — в разделе Настройки → Интерфейсы появился блок с чекбоксами для привязки nfqws2 к конкретным интерфейсам (с отображением состояния UP/DOWN и IP-адресов), пункт «Все интерфейсы» для работы на всех сразу
- Унифицирована версия GUI (единый источник `v0.13.0` во всех местах)

### Изменено
- Новые API-эндпоинты: `GET /api/interfaces`, `POST /api/interfaces/select`

## v0.12.0 — Фаза 12: Autocomplete (синтаксис-помощник)

### Добавлено
- **Autocomplete для стратегий** — выпадающий popup с контекстными подсказками при вводе аргументов
  - Автоматический вызов при наборе `--` (начало флага)
  - Ручной вызов по `Ctrl+Space` в любой позиции
  - Контекстные подсказки: флаги, desync-методы, L7-протоколы, fooling, позиции, TLS-модификации
  - Подсказки субпараметров lua-цепочек (`blob=`, `pos=`, `repeats=`, …)
  - Подсказки файлов: блобы, hostlists, ipsets с сервера
  - Фильтрация по мере набора
  - Навигация стрелками ↑↓, вставка Enter/Tab, закрытие Escape
  - Цветовые иконки по категориям: 🔵 filter, 🔴 desync, 🟢 list, 🟡 global, 🟠 dup
  - Краткие описания параметров справа от каждого варианта
  - Позиционирование popup под курсором через mirror-div
  - Хинт `Ctrl+Space` в textarea
- **Новый модуль** — `web/js/utils/autocomplete.js` (IIFE `NfqwsAutocomplete`)
- **CSS-стили** — `.nfq-ac-*` для popup, адаптив для мобильных

### Интеграция
- Autocomplete подключается при открытии редактора стратегий
- Переподключается при добавлении/удалении профилей
- Очищается при закрытии модала и уходе со страницы

## v0.11.0 — Фаза 11: Интеграция и полировка

### Добавлено
- **Подсветка синтаксиса стратегий** — цветовое выделение параметров nfqws/nfqws2
  - Флаги (`--filter-tcp`, `--dpi-desync`, `--lua-desync`) — голубые
  - Desync-методы (`fake`, `multisplit`, `fakedsplit`) — красные, жирные
  - Субпараметры-ключи (`blob`, `pos`, `repeats`) — зелёные
  - Субпараметры-значения и fooling — оранжевые
  - L7-протоколы (`http`, `tls`, `quic`) — фиолетовые
  - Файлы (`.txt`, `.bin`) — голубые с подчёркиванием
  - Числа и позиции — свои цвета
  - Разделитель `--new` — оранжевый с фоном
- **Синтаксис в превью стратегий** — подсветка в модальном окне превью команды
- **Синтаксис в карточках стратегий** — превью аргументов каждого профиля
- **Синтаксис в логах** — автоопределение и подсветка nfqws-команд в логах
- **Синтаксис в dashboard и control** — подсветка nfqws-аргументов в мини-логах
- **Утилиты** — `web/js/utils/syntax.js` (подсветка), `web/js/utils/debounce.js` (оптимизация)
- **README.md** — полная документация: установка, использование, API, FAQ
- **CHANGELOG.md** — история версий
- Поддержка обоих стилей параметров: nfqws2 (lua-desync) и classic nfqws (dpi-desync)
- Справочник 60+ параметров nfqws с описаниями и допустимыми значениями
- Готовая инфраструктура для будущего autocomplete-помощника
