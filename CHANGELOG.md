## v0.13.3 — Привязка firewall к WAN-интерфейсам

### Исправлено
- **Firewall не привязывался к WAN-интерфейсам** — правила iptables/nftables применялись без `-o $IFACE_WAN`, перехватывая трафик на всех интерфейсах. Теперь `firewall.py` читает `interfaces.wan` и `interfaces.wan6` из конфига и создаёт правила с привязкой к конкретным интерфейсам (как в zapret2 `_fw_nfqws_post4`/`_fw_nfqws_post6`)

### Добавлено
- **Раздельные IPv4/IPv6 правила** — iptables для IPv4, ip6tables для IPv6. IPv6 отключается через `nfqws.disable_ipv6`
- **Авто-определение WAN при старте firewall** — если `interfaces.wan` пуст, определяется по `/proc/net/route` (fallback на `ip route`)
- **Fallback WAN6 → WAN** — если `interfaces.wan6` не задан и не определён, используется значение WAN (как `IFACE_WAN6` в zapret2)
- **Множественные WAN-интерфейсы** — для каждого интерфейса создаётся отдельное правило в iptables (`-o eth0`, `-o eth1`); в nftables используется `oifname { "eth0", "eth1" }`
- **Очистка ip6tables** — при снятии правил теперь удаляются и правила ip6tables

### Изменено
- `_apply_iptables()` принимает `wan4_ifaces` и `wan6_ifaces`
- `_apply_nftables()` добавляет `oifname` фильтр
- Логирование при старте показывает привязанные интерфейсы
- Унифицирована версия GUI (`v0.13.3` во всех модулях)

## v0.13.2 — Исправление интерфейсов

### Исправлено
- **Чекбоксы интерфейсов не работали** — полностью переработан блок интерфейсов: убран нерабочий механизм `bind_to` с чекбоксами, который вызывал двойное срабатывание (конфликт `onclick` на label + `onchange` на input) и блокировку через `pointer-events:none`
- **WAN/WAN6/LAN не заполнялись автоматически** — добавлено авто-определение ролей по таблице маршрутов, аналогично логике zapret2

### Добавлено
- **Назначение ролей через бейджи** — вместо чекбоксов теперь у каждого интерфейса кликабельные бейджи WAN / WAN6 / LAN; клик назначает/снимает роль, текстовые поля WAN/WAN6/LAN обновляются автоматически
- **Авто-определение WAN/WAN6/LAN** — WAN по default route (`/proc/net/route` + `ip route`), WAN6 по `ip -6 route` с fallback на WAN, LAN по bridge-интерфейсу или приватной подсети RFC 1918
- **Кнопка «Определить автоматически»** — заполняет поля WAN/WAN6/LAN из таблицы маршрутов
- **Кнопка «Сбросить роли»** — очищает все назначения
- Placeholder полей WAN/WAN6/LAN показывает авто-определённое значение
- Поле `detected` в ответе `GET /api/interfaces`

### Удалено
- Эндпоинт `POST /api/interfaces/select` и конфиг-ключ `interfaces.bind_to` — не использовались в zapret2
- Чекбоксы «Все интерфейсы» / выбор конкретных — заменены бейджами ролей

### Изменено
- Формат полей WAN/WAN6/LAN — через пробел (как `IFACE_WAN` в zapret2), например `eth0 eth1`
- Унифицирована версия GUI (`v0.13.2` во всех модулях)

  - Краткие описания параметров справа от каждого варианта
  - Позиционирование popup под курсором через mirror-div
  - Хинт `Ctrl+Space` в textarea
